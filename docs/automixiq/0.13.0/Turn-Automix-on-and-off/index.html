    Turn AutomixIQ on and off¶

    AutomixIQ's player can be turned on or off after initialisation. You need to turn on the player before playing any music.

    Note

    If there is another player currently playing music, we suggest pausing the playback once AutomixIQ's player is turned on to prevent competing for audio control. When AutomixIQ's player is turned off, any other player can be resumed.

    Turn AutomixIQ on¶

    You can turn on the player without any tracks, or turn on the player with a list of tracks and its metadata.

    Note

    If you start the player without tracks, tracks will need to be provided via the playback control methods later, see Load tracks.

    Turn on without tracks
    AutomixSDK.getInstance()?.automixOn(
    onComplete = { automixSDKStatus -> 
        // Player is turned on and is waiting for tracks to be loaded.
    }, onError = { automixSDKError, automixSDKStatus ->
        // Something went wrong, playback did not start
    }
)

    Note

    Please make sure you adhere to the Audio file requirements before loading any tracks to AutomixIQ

    Turn on with tracks
    // list of track ids
val electronicMusicPlaylist = listOf("116484855","66906273","120584575","116887896")

// list of `AutomixSDKTrackModel` for tracks
val cachedMetadata = listOf(trackMetadata1, trackMetadata2, trackMetadata3, trackMetadata4) 

AutomixSDK.getInstance()?.automixOn(
    identifiers = electronicMusicPlaylist,
    metadata = cachedMetadata,      // nullable, if null and the SDK will call `getTrackMetadata` in `TunedMusicCacheHook` to get metadata
    playTrackIndex = 0,             // start playback index in playlist
    seconds = 0,                    // start playback seconds in track
    autoplay = true,                // start playback immediately or not
    contextType = "playlist",       // for reporting purposes
    contextId = "playlist-id",      // for reporting purposes
    onComplete = { automixSDKStatus -> 
        // Player is turned on and will start playback if autoplay is true
    }, onError = { automixSDKError, automixSDKStatus ->
        // Something went wrong, playback did not start
    }
)

    Turn AutomixIQ off¶

    Turning off AutomixIQ's player will stop playback

    AutomixSDK.getInstance()?.automixOff(
    onComplete = { automixSDKStatus -> 
        // Playback should be stopped by now, resume other players if needed.
        // `automixSDKStatus` contains the last status of the player, can be used to provide information to other players to resume playback.
    },
    onError = { automixSDKError, automixSDKStatus ->
        // Something went wrong, playback did not stop
    }
)

    For more information about AutomixSDKStatus, see Update UI.

    Check on/off state¶

    To check whether AutomixIQ's player is turned on

    AutomixSDK.getInstance()?.on()
