    Configure the SDK with custom backend¶

    Overview¶

    This document explains how to integrate AutomixIQ with your application you are using a custom backend to access tracks and metadata.

    To initialise AutomixIQ when you take the responsibility of providing audio data and metadata you need:

    • A store ID and an application ID provided by the Tuned Global team
    • An implementation of the AutomixSDKStreamingListener interface

    Implement your AutomixSDKStreamingListener¶

    AutomixSDKStreamingListener is an interface that provides AutomixIQ with the essential information from your service to support playback. Extend the AutomixSDKStreamingListener class and implement the required functions as shown below:

    • isLoggedIn
    • asyncCheckDeviceStatus
    • getDeviceIdentifier
    • syncGetTrackMetadata
    • syncGetCachedTrack
    • cacheMetadata
    • asyncGetTrackMetadata
    • asyncGetStream
    • onTrackDownloadCompeleted
    • onTrackDownloadProgressChanged

    You can find a detailed explanation in API Reference

    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {

    // Return whether the user is logged in
    override fun isLoggedIn() {
        return this.loggedIn
    }

    // Query and inform whether the device is allowed to continue playing by calling onComplete/onError
    override fun asyncCheckDeviceStatus(onComplete: ((Boolean) -> Unit)?,
                                        onError: ((AutomixSDKError) -> Unit)?) -> Unit) {
        this.canDeviceContinuePlaying() { allowedToPlay ->
            onComplete(allowedToPlay)
        }
    }

    // Return this device's unique identifier
    override fun getDeviceIdentifier(): String? {
        return this.deviceIdentifier
    }

    // Return the track metadata if available offline or cached
    override fun syncGetTrackMetadata(trackId: String?): AutomixSDKTrackModel? {
        // If this track is cached, return its metadata
        if (!this.isTrackCached(trackId)) {
            return null
        }
        return this.metadataForTrack(trackId)
    }

    // Return the track data if available offline or cached 
    override fun syncGetCachedTrack(trackId: String): ByteArray? {
        // If this track is cached, return its data
        if (!this.isTrackCached(trackId)) {
            return null
        }
        return this.unencryptedDataForTrack(trackId)
    }

    // Cache this metadata for later use
    override fun cacheMetadata(metadata: List<AutomixSDKTrackModel>) {
        this.metadataForTrack[metadata.trackId] = metadata
    }

    // Download the track metadata and return it to AutomixIQ by running onComplete/onError
    override fun asyncGetTrackMetadata(trackId: String,
                                       onComplete: ((AutomixSDKTrackModel) -> Unit)?,
                                       onError: ((AutomixSDKError) -> Unit)?) -> Unit) {
        this.getTrackMetadata(trackId) { error, trackMetadata ->
            val automixSDKTrackModel = translateMetadata(trackMetadata)
            if (error != null) {
                onError(error)
                return@getTrackMetadata
            }
            onComplete(automixSDKTrackModel)
            return@getTrackMetadata
        }
    }

    // Retrieve the track's url, and return it to AutomixIQ by running onComplete/onError
    override fun asyncGetStream(streamingDeviceId: String, sessionId: String, trackId: String, 
                                onComplete: ((String /* URL */, String? /* newSessionId */) -> Unit)?,
                                onError: ((AutomixSDKError) -> Unit)?) {
        this.getTrackURL(streamingDeviceId, sessionId, trackId) { error, url, newSessionId ->
            if (error != null) {
                onError(error)
                return@getTrackURL
            }
            onComplete(url, newSessionId, null)
        }
    }

    // AutomixIQ will return the audio data once a download is completed
    override fun onTrackDownloadCompeleted(trackId: String, data: ByteArray) {
        this.cacheTrack(trackId, data)
    }

    // Monitor the download progress (0 to 1)
    override fun onTrackDownloadProgressChanged(trackId: String, progress: Float) {
        this.updateTrackDownloadProgress(trackId, progress)
    }
}

    Do keep in mind that AutomixIQ downloads tracks lazily as it requires data for analysis / playback. There is no guarantee a track will be fully downloaded if the track is not played in its entirety.

    Initialise AutomixSDK instance¶

    Create an AutomixSDKConfig first. You can get the store ID, application ID from the Tuned Global team.

    val automixConfig = 
    AutomixSDKConfig.Builder()
        .apiKey(myAPIKey)
        .country(myCountryISO2Code) // Device country code e.g. "US", "AU"
        .storeId(myTunedStoreId )
        .applicationId(myTunedApplicationId)
        .logLevel(AutomixSDKLogLevel.DEBUG)
        .build()

    Initialise the AutomixSDK instance with the AutomixSDKStreamingListener and AutomixSDKConfig created in previous steps. We recommend initialising the SDK once your music background service is up and running.

    Note

    AutomixIQ requires an active internet connection for its initial verification. After this initial setup, it can be initialised offline. However, note that there is a grace period of 7 days, after which an internet connection will be required again for re-verification.

        AutomixSDK.initialise(applicationContext,
                          automixSDKStreamingListener,
                          config,
                          onComplete = 
    { _ -> 
        // The Automix SDK public functions can be used from this point onwards :)

        // We can add a status listener to get UI updates and play events
        val automixSDKStatusListener: AutomixSDKStatusListener = myAutomixSDKStatusListener
        AutomixSDK.getInstance()?.addStatusListener(automixSDKStatusListener)

        // ...
    }, onError = { error, _ ->
        // This means you are forbidden to use the Automix SDK or you have no internet connection.
    })

    Implement AutomixSDKStatusListener (Optional)¶

    The AutomixSDKStatusListener is responsible for updating your player’s UI, please refer to Update UI

    Check initialised state¶

    To check whether AutomixIQ is initialised 

    AutomixSDK.getInstance()?.initialised()

    Shut down¶

    To shut down AutomixIQ

    AutomixSDK.terminate()

    After you have terminated the SDK, you will need to reinitialise if you want to use AutomixIQ again.
