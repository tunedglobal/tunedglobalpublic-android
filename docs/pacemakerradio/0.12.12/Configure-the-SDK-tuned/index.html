    Configure the SDK with Tuned Global backend¶

    Overview¶

    This document explains how to integrate AutomixIQ with your application if it is already using Tuned Global’s backend to access tracks and metadata.

    To initialise AutomixIQ with Tuned Global's API and services, you need

    • A store ID and an application ID provided by the Tuned Global team
    • An implementation of the TunedRestHook interface

    Implement your TunedRestHook¶

    TunedRestHook is an interface that facilitates communication between the Tuned Global Backend and the AutomixIQ SDK regarding the user's access token and device information. Extend the TunedRestHook class and implement the required functions as shown below:

    • getAuthenticatedDevice
    • getUniqueDeviceIdentifier
    • onTunedRestTokenExpired

    You can find a detailed explanation in API Reference

    class MyTunedRestHook: TunedRestHook {

    // Return the Tuned API device ID integer
    override suspend fun getAuthenticatedDevice(uniqueDeviceIdentifier: String): String? {
        val myTunedDeviceId: String = tunedDeviceId
        return myTunedDeviceId
    }

    // Return an ID that can be used to identify a device.
    // This is typically a randomly generated UUID that persists through app restarts.
    override fun getUniqueDeviceIdentifier(): String {
        val uuid: String = myUniqueDeviceIdentifier
        return uuid
    }

    // Use the info provided from the parameters to refresh the Tuned user token
    override suspend fun onTunedRestTokenExpired(uniqueDeviceIdentifier: String,
                                                 countryCode: String,
                                                 tunedUserId: String?,
                                                 oldAccessToken: String?,
                                                 refreshToken: String?,
                                                 tokenType: String?): TunedAuthenticationResponse? {
        // your implementation of refreshing Tuned user token
        val tunedAuthenticationResponse: TunedAuthenticationResponse? = null
        withContext(Dispatchers.IO) {
            suspendCancellableCoroutine { continuation ->
                refreshToken(refreshToken, tokenType, uniqueDeviceIdentifier) { authToken, tokenType, tokenExpiration, refreshToken ->
                    val userId = tunedUserId
                    tunedAuthenticationResponse = TunedAuthenticationResponse(authToken,
                                                                              tokenType,
                                                                              tokenExpiration,
                                                                              refreshToken,
                                                                              tunedUserId)
                    continuation.resume(Unit)
                }
            }
        }
        return tunedAuthenticationResponse
    }
}

    Implement your RadioUIListener¶

    RadioUIListener is an interface that provides data required for the Radio UI to function, as well as being responsible for presenting UI PacemakerRadio is not able to do on its own. Extend the RadioUIListener class and implement the required (an optionally, those marked as optinal) methods as shown below:

    • presentTracklist
    • presentAuthentication
    • presentProfile
    • isTrackLiked
    • setTrackLiked
    • applicationContext

    and optionally,

    • willDismissRadioUI

    • didDismissRadioUI

    • willCloseRadioUI

    • didCloseRadioUI
    class MyRadioUIListener: RadioUIListener {

    override fun presentTracklist(context: Context, completion: (tracks: List<AutomixSDKTrackModel>?, playlistName: String?) -> Unit) {
        // Your implementation for presenting UI that allows to select one or more tracks:
        // - Feed the track/tracks selected as the second paramenter of the completion block,
        // - If the tracks selected are part of a playlist, feed the name opf that playlist as the thirsd parameter of the completion block.
        val context = myActivityContext
        val myTracklist = MyTracklist(context = myActivityContext, selectedTracksBloc = { tracks, playlistName ->
            completion(track, playlist, playlistName)
        })
        myTracklist.show()
    }

    override fun presentAuthentication(context: Context, completion: (success: Boolean) -> Unit) {
        // At a time where PacemakerRadio requires a Tuned User Id, if it does not have one it will require the user to authenticate by calling this function.
        // Present your login block here
        val context = myActivityContext
        val myLoginDialog = MyLoginDialog(context = myActivityContext, onLogin = { success -> 
            completion(success)
        })
        myLoginDialog.show()
    }

    override fun presentProfile(context: Context, identifier: String?, completion: () -> Unit) {
        // At times, PacemakerRadio will need to present a users' profile. It will do so by calling this function.
        val context = myActivityContext
        val myProfileDialog = MyProfileDialog(context = myActivityContext)
        myProfileDialog.show()
    }

    override fun isTrackLiked(identifier: String, completion: (isLiked: Boolean) -> Unit) {
        // At times, PacemakerRadio will need to know if a track is part of the user's "Liked" playlist.
        // It will ask the client application to do so by calling this method.
        // You can take your time to load the liked playlist if you do not have it around in order to respond.
        // You can respond synchronously if you do know
        val likedPlaylist = myLikedPlaylist
        completion(myLikedPlaylist.contains(identifier))
    }

    override fun setTrackLiked(
            identifier: String,
            liked: Boolean,
            completion: (error: RadioSDKError?, success: Boolean) -> Unit) {
        // PacemakerRadio allows for a user to flag/unflag a track as Liked. When that happens, it will call this method.
        val likedPlaylist = myLikedPlaylist
        if(liked) {
            likedPlaylist.add(identifier)
        } else {
            likedPlaylist.remove(identifier)
        }
    }

    override fun applicationContext(): Context {
        // At times PacemakerRadio requires the application context to look up resources
        return applicationContext
    }

    Initialise RadioSDK instance¶

    Create an RadioSDKConfig first. The store and application ID will be provided by the Tuned Global team. You can tweak the PacemakerRadio UI and available feature set by changing flags on the cofing object.

    val radioConfig =
    RadioSDKConfig.Builder()
        .apiKey(myAPIKey)
        .country(myCountryISO2Code) // Device country code e.g. "US", "AU"
        .storeId(myTunedStoreId )
        .applicationId(myTunedApplicationId)
        .logLevel(AutomixSDKLogLevel.DEBUG)
        .build()

    Initialise the RadioSDK instance with the TunedRestHook, RadioUIListener and AutomixSDKConfig created in previous steps. We recommend initialising the SDK once your music background service is up and running.

    Note

    PacemakerRadio requires an active internet connection for its initial verification. After this initial setup, it can be initialised offline. However, note that there is a grace period of 7 days, after which an internet connection will be required again for re-verification.

        RadioSDK.initialise(applicationContext = applicationContext,
                        streamingListener = myTunedRestHook,
                        config = radioConfig,
                        radioUIListener = radioUIListener,
                        onComplete = { automixStatus ->

        // The PacemakerRadio public functions can be used from this point onwards :)

        // We can set a music cache hook if we want to support offline playback
        val musicCacheHook: TunedMusicCacheHook = myMusicCacheHook
        RadioSDK.getInstance()?.setMusicCacheHook(musicCacheHook)

        // We can add a status listener to get UI updates and play events, you can add as many AutomixSDKStatusListener as you like
        val automixSDKStatusListener: AutomixSDKStatusListener = myAutomixSDKStatusListener
        RadioSDK.getInstance()?.addStatusListener(automixSDKStatusListener)

        // ...
    }, onError = { automixSDKError, automixSDKStatus -> 
        // This means you do not have permission to use AutomixIQ or there is no internet connection when required.
        return
    })

    Implement TunedMusicCacheHook (Optional)¶

    PacemakerRadio supports offline playback using local music data. You can implement the TunedMusicCacheHook to

    • provide track metadata and cached audio data,
    • extract audio data downloaded by PacemakerRadio
    • monitor the download progress

    This approach enhances efficiency and prevents PacemakerRadio from having to request this data from Tuned Global’s API if they are already available. This also enables offline playback when all the required data is available locally.

    Implement the TunedMusicCacheHook and add it to the SDK after initialisation. You can find a detailed explanation in API Reference

    SetMusicCacheHook
    RadioSDK.getInstance()?.setMusicCacheHook(MyMusicCacheHook())
    MyMusicCacheHook
    public class MyMusicCacheHook: TunedMusicCacheHook {

    // Provide the track metadata. For offline playback, this is required if the metadata is not loaded when playing tracks.
    override fun getTrackMetadata(trackId: String?): AutomixSDKTrackModel? {
        // If the track is not cached, return null and PacemakerRadio will try to fetch it.
        if (!isTrackCached(trackId)) {
            return null
        }
        return metadataForTrack(trackId)
    }

    // Provide the cached audio data. For offline playback, this is required.
    override fun getCachedTrack(trackId: String): ByteArray? {
        // If the track is not cached, return null and PacemakerRadio will try to stream it.
        if (!isTrackCached(trackId)) {
            return null
        }

        return rawAudioDataForTrack(trackId)
    }

    // PacemakerRadio will 
    // - stream and download the tracks when they are not provided from `getCachedTrack` and call this method once the download for each track has completed
    // - use the cached audio data from getCachedTrack and call this method once the data provided is verified for each track
    override fun onTrackDownloadCompeleted(trackId: String, data: ByteArray) {
        cacheTrack(trackId, data)
    }

    // Monitor the download progress (0 to 1)
    override fun onTrackDownloadProgressChanged(trackId: String, progress: Float) {
        updateTrackBufferingProgress(trackId, progress)
    }
}

    Note

    If the byte data for a track provided through getCachedTrack does not fit the Audio file requirements. PacemakerRadio will try to stream the track from the connected backend.

    Check initialised state¶

    To check whether PacemakerRadio is initialised 

    RadioSDK.getInstance()?.initialised()

    Shut down¶

    To shut down PacemakerRadio

    RadioSDK.terminate()

    After you have terminated the SDK, you will need to reinitialise if you want to use PacemakerRadio again.
