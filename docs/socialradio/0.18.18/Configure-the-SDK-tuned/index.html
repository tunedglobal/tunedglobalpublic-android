    Configure the SDK with Tuned Global backend¶

    Overview¶

    This document explains how to integrate SocialRadio with your application using Tuned Global’s backend to access tracks and metadata.

    To initialise SocialRadio with Tuned Global's API and services, you need

    • Your Tuned Global store ID and application ID (provided by the Tuned Global team)
    • An implementation of the TunedRestHook interface

    Implement your TunedRestHook¶

    TunedRestHook is an interface that facilitates communication between the Tuned Global Backend and the SocialRadio SDK regarding the user's access token and device information. Extend the TunedRestHook class and implement the required functions as shown below:

    • getAuthenticatedDevice
    • getUniqueDeviceIdentifier
    • onTunedRestTokenExpired
    • automaticallyLogPlayback

    You can find a detailed explanation in API Reference

    class MyTunedRestHook: TunedRestHook {

    // Return the Tuned API device ID integer
    override suspend fun getAuthenticatedDevice(uniqueDeviceIdentifier: String): String? {
        val myTunedDeviceId: String = tunedDeviceId
        return myTunedDeviceId
    }

    // Return an ID that can be used to identify a device.
    // This is typically a randomly generated UUID that persists through app restarts.
    override fun getUniqueDeviceIdentifier(): String {
        val uuid: String = myUniqueDeviceIdentifier
        return uuid
    }

    // Use the info provided from the parameters to refresh the Tuned user token
    override suspend fun onTunedRestTokenExpired(uniqueDeviceIdentifier: String,
                                                 countryCode: String,
                                                 tunedUserId: String?,
                                                 oldAccessToken: String?,
                                                 refreshToken: String?,
                                                 tokenType: String?): TunedAuthenticationResponse? {
        // your implementation of refreshing Tuned user token
        val tunedAuthenticationResponse: TunedAuthenticationResponse? = null
        withContext(Dispatchers.IO) {
            suspendCancellableCoroutine { continuation ->
                refreshToken(refreshToken, tokenType, uniqueDeviceIdentifier) { authToken, tokenType, tokenExpiration, refreshToken ->
                    val userId = tunedUserId
                    tunedAuthenticationResponse = TunedAuthenticationResponse(authToken,
                                                                              tokenType,
                                                                              tokenExpiration,
                                                                              refreshToken,
                                                                              tunedUserId)
                    continuation.resume(Unit)
                }
            }
        }
        return tunedAuthenticationResponse
    }

    // Return True to allow the SDK handle playback events required by Tuned Global, or false to implement those events yourself.
    override fun automaticallyLogPlayback(): Boolean {
        return true // It will log events automatically
    }
}

    Implement your SocialRadioRadioUIListener¶

    SocialRadioRadioUIListener is an interface that provides data required for the Radio UI to function, as well as being responsible for presenting UI SocialRadio is not able to do on its own. Extend the SocialRadioRadioUIListener class and implement the required (an optionally, those marked as optinal) methods as shown below:

    • presentTracklist
    • presentAuthentication
    • presentProfile

    and optionally,

    • onDismiss
    • onDestroy
    class MyRadioUIListener: SocialRadioRadioUIListener {

    override fun presentTracklist(context: Context, completion: (tracks: List<AutomixSDKTrackModel>?, 
                                                                 contextType: String?, contextId: Int,
                                                                 playlistName: String?) -> Unit) {
        // Your implementation for presenting UI that allows to select one or more tracks:
        // - Feed the track/tracks selected as the second parameter of the completion block,
        // - If the tracks selected are part of a playlist, feed the name of that playlist as the third parameter of the completion block.
        val context = myActivityContext
        val myTracklist = MyTracklist(context = myActivityContext, selectedTracksBloc = { tracks, playlistName ->
            completion(track, playlist, playlistName)
        })
        myTracklist.show()
    }

    override fun presentAuthentication(context: Context, completion: (success: Boolean) -> Unit) {
        // When SocialRadio needs a Tuned User ID and doesn’t have one, it calls this function to prompt the user to authenticate.
        // Present your login block here
        val context = myActivityContext
        val myLoginDialog = MyLoginDialog(context = myActivityContext, onLogin = { success -> 
            completion(success)
        })
        myLoginDialog.show()
    }

    override fun presentProfile(context: Context, identifier: String?, completion: () -> Unit) {
        // At times, SocialRadio will need to present a users' profile. It will do so by calling this function.
        val context = myActivityContext
        val myProfileDialog = MyProfileDialog(context = myActivityContext)
        myProfileDialog.show()
    }

    Initialise SocialRadio instance¶

    Create an SocialRadioConfig first. The store and application ID will be provided by the Tuned Global team. You can tweak the SocialRadio UI and available feature set by changing flags on the config object.

    val radioConfig =
    SocialRadioConfig.Builder()
        .apiKey(myAPIKey)
        .country(myCountryISO2Code) // Device country code e.g. "US", "AU"
        .storeId(myTunedStoreId )
        .applicationId(myTunedApplicationId)
        .build()

    Initialise the SocialRadio instance with the TunedRestHook, SocialRadioRadioUIListener and SocialRadioConfig created in previous steps. We recommend initialising the SDK once your music background service is up and running.

    Note

    SocialRadio requires an active internet connection for its initial verification; it can then run offline for up to 7 days, after which an internet connection is required again for re-verification.

        SocialRadio.initialise(applicationContext = applicationContext,
                           tunedRestHook = myTunedRestHook,
                           config = radioConfig,
                           radioUIListener = radioUIListener,
                           onComplete = { automixStatus ->

        // The SocialRadio public functions can be used from this point onwards :)

        // We can add a status listener to get UI updates and play events, you can add as many AutomixSDKStatusListener as you like
        val socialRadioStatusListener: SocialRadioStatusListener = mySocialRadioStatusListener
        SocialRadio.getInstance()?.addStatusListener(socialRadioStatusListener)

        // ...
    }, onError = { socialRadioError, socialRadioStatus -> 
        // This means you do not have permission to use SocialRadio or there is no internet connection when required.
        return
    })

    Implement SocialRadioStatusListener (Optional)¶

    The SocialRadioStatusListener is responsible for updating your player’s UI, please refer to Update UI

    Check initialised state¶

    To check whether SocialRadio is initialised 

    SocialRadio.getInstance()?.initialised()

    Shut down¶

    To shut down SocialRadio

    SocialRadio.terminate()

    After you have terminated the SDK, you will need to reinitialise if you want to use SocialRadio again.
